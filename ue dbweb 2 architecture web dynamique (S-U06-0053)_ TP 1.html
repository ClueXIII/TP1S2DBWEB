TP 1

Sujet no 1 : Formulaires HTML

Pour la reprise, n'hésitez pas à repasser rapidement les TP du module DBWeb du S1 pour vous remettre en tête les principaux concepts. Pour l'édition, des éditeurs de texte sont généralement disponibles par défaut (par exemple gedit sur Linux, ou notepad sur Windows...). Mais vous pouvez aussi recourir à des éditeurs plus complets pour la programmation : par exemple Atom (version portable téléchargeable ici : https://github.com/atom/atom/releases/download/v1.24.0/atom-amd64.tar.gz), Sublim Text, BBEdit...

1. Zone de Formulaire

Une zone de formulaire est une zone où l'utilisateur peut entrer des informations qui seront transmises selon une méthode choisie (GET ou POST) lorsque l'utilisateur appuie sur un bouton spécifique de soumission. Un formulaire peut alors contenir différents type de conteneurs pour la saisie des données.

Exemple :

A faire :


Créer une page web comportant un titre, et un formulaire avec deux boutons seulement pour commencer : un pour la soumission et un pour la réinitialisation.

2. Zones de texte

Les zones de texte d'un formulaire permettent d'enregistrer dans des variables (ou champs) des informations entrées par l'utilisateur. Les zones de texte peuvent être libres, ou codées (pour les mots de passe), ou elles peuvent avoir une taille contrainte (avec plusieurs lignes).

Exemple :

A faire :

Dans le formulaire créé précédemment, introduire plusieurs champs texte :
  1. nom, prénom, age, e-mail (champs textes simples). Pour le champ e-mail placer un texte de suggestion "nom.prenom@univ-avignon.fr" pour guider l'utilisateur.
  2. mot de passe et sa confirmation (codées),
  3. et une zone de commentaires libres sur 3 lignes,
  4. puis régler la méthode sur GET pour observer les valeurs saisies lors de la soumission.
  5. Tester avec le bouton de soumission.

Lien vers la page effectuée

Rappel : Lors de la création du formulaire, deux méthodes d'envoi sont possibles : GET ou POST. POST est la valeur qui correspond à un envoi de données stockées dans le corps de la requête, tandis que GET correspond à un envoi des données codées dans l'URL, et séparées de l'adresse du script de destination par un point d'interrogation. On ne pourra tester  POST que lorsque que l'action du formulaire désignera une page comportant un script permettant d'accéder aux données transmises (d'ici quelques TP).

3. Choix multiples

D'autres balises permettent de réaliser une saisie de variables prédéfinies dans une liste d'options. Le premier type est le bouton radio et case à cocher. Les boutons radios permettent de choisir au plus un élément parmi plusieurs. Les cases à chocher permettent de sélectionner autant d'éléments que l'on souhaite.

Exemple :

A faire :

Dans le formulaire, insérer :
  1. des boutons radios pour demander le genre de l'utilisateur (1 seul choix possible),
  2. des cases à cocher pour obtenir les sports pratiqués (au moins 6, l'utilisateur en choisit autant qu'il veut).

4. Listes à choix multiples

Les listes à choix multiples sont l'équivalent des options vues précédemment, mais dans un autre mode de présentation. Les listes déroulantes limitent le choix à une option, tandis que les listes libres permettent d'en sélectionner 0 ou plusieurs. De même de nombreuses options d'affichages permettent de s'adapter à chaque situation pour améliorer l'expérience utilisateur.

Exemple :

A faire :

Insérer dans le formulaire :
  1. une liste déroulante de choix d'activité principal, avec le 2e élément sélectionné par défaut,
  2. une liste de choix de moyens de transport pour se rendre au stage.
